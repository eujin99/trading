# KIS 급등주 자동매매 봇 사용 매뉴얼

현재 코드는 **시간 자동 실행이 아닌 메뉴 수동 실행 방식**입니다.
원하는 작업을 메뉴에서 직접 선택해 진행합니다.

## 1) 구성 파일

```text
files/
├── config.py         # 공통 키/계좌/기본 파라미터
├── config_kr.py      # 국장 전용 설정 오버라이드
├── config_us.py      # 미장 전용 설정 오버라이드
├── kis_api.py        # KIS 인증/시세/주문 API 래퍼
├── autotrader.py     # 공통 실행 엔진
├── autotrader_kr.py  # 국장 전용 실행 파일
├── autotrader_us.py  # 미장 전용 실행 파일
└── README.md         # 이 문서
```

## 2) 실행 전 준비

### 2-1. 패키지 설치

```bash
pip install requests
```

### 2-2. 암호화 키 설정 (`TRADING_MASTER_KEY`)

`config.py`, `config_acc2.py`의 민감정보는 `ENC(...)` 암호문으로 저장됩니다.
실행 전에 아래처럼 마스터 키를 로드해야 합니다.

```bash
# 프로젝트 루트(/Users/eujin/trading) 기준
source .secrets.key
```

또는 셸 프로필에 직접 등록해도 됩니다.

```bash
export TRADING_MASTER_KEY="여기에_본인_마스터_키"
```

새 민감값을 암호화할 때:

```bash
python encrypt_secret.py "평문값"
```

### 2-3. `config.py` 필수 설정

```python
APP_KEY    = "발급받은_APP_KEY"
APP_SECRET = "발급받은_APP_SECRET"
ACCOUNT_NO = "계좌번호-상품코드"   # 예: 50123456-01
IS_VIRTUAL = True                  # 모의투자=True, 실전=False
```

중요:
- `ACCOUNT_NO`는 `계좌번호-상품코드` 형식 권장 (예: `50123456-01`)
- `IS_VIRTUAL=True`에서 충분히 검증 후 실전 전환

## 3) 실행 방법

기본 실행은 아래 하나만 사용하면 됩니다.

```bash
python autotrader.py
```

계좌를 2개 병렬로 돌릴 때는 계좌2 전용 설정(`config_acc2.py`)을 만들고 아래처럼 각각 실행합니다.

```bash
# 계좌1 (기본 config.py)
python autotrader.py

# 계좌2 (config_acc2.py)
python autotrader_acc2.py
```

메뉴 `1` 또는 `3` 실행 시 `1.국장 / 2.미장`을 선택하면 해당 시장 설정(`config_kr.py`, `config_us.py`)이 자동 적용됩니다.
`autotrader_kr.py`, `autotrader_us.py`는 보조 실행 파일이며 필수는 아닙니다.
실행하면 아래 메뉴가 표시됩니다.

```text
1) 스크리닝 실행 (1.국장/2.미장 선택)
2) 최근 스크리닝 결과 매수
3) 스크리닝 후 즉시 매수 (1.국장/2.미장 선택)
4) 보유 종목 점검 및 조건 매도
5) 보유 종목 상태 보기
6) 전량 강제 매도
9) 종료
```

## 4) 메뉴별 사용법

### 1) 스크리닝 실행
- 실행 시 `1.국장 / 2.미장`을 먼저 선택
- 보유 종목이 있을 때 다른 시장으로 변경은 불가
- 국장: 거래량 상위/등락률 상위 기반 후보 생성
- 미장: `US_UNIVERSE` 티커 풀 기반 후보 생성
- 후보 종목을 점수화 후 상위 `MAX_STOCKS`개 출력
- 이 결과는 `최근 스크리닝 결과`로 메모리에 저장됨

### 2) 최근 스크리닝 결과 매수
- 직전에 실행한 스크리닝 결과를 기준으로 매수 실행
- 스크리닝을 먼저 안 했다면 실행되지 않음

### 3) 스크리닝 후 즉시 매수
- 실행 시 `1.국장 / 2.미장`을 먼저 선택한 뒤 스크리닝과 매수를 한 번에 실행
- 가장 빠른 실전 사용 루트

### 4) 보유 종목 점검 및 조건 매도
- 현재 보유 종목별 실시간 가격을 조회해 자동 매도 조건 체크
- 손절가 하회 시 즉시 손절하지 않고, `STOP_BREACH_CONFIRM_COUNT`만큼 연속 확인
- 최초 손절 구간 진입 시 1회 물타기 시도 (조건 충족 시)
- 물타기 후에는 새 평단 기준 `POST_DCA_STOP_LOSS_RATE` 손절 적용
- 1차 목표가 이상(미실행 시): 절반 매도
- 2차 목표가 이상: 전량 매도

주의:
- 현재 구조는 **자동 주기 점검이 아님**
- 익절/손절 반영하려면 메뉴 4를 직접 실행해야 함

### 5) 보유 종목 상태 보기
- 봇 메모리에 저장된 보유 상태(수량/평단/목표가/손절가) 출력

### 6) 전량 강제 매도
- 포트폴리오에 있는 종목을 전량 시장가 매도
- 장 마감 자동 발동 없음, 필요 시 직접 실행

### 9) 종료
- 보유 종목이 남아 있으면 종료 전 강제 매도 여부 확인

## 5) 매수 금액 산정 방식 (퍼센트 기반)

매수 수량은 종목별 주문 시점에 아래 순서로 계산됩니다.

1. `inquire-psbl-order`로 해당 종목 주문 가능 현금 조회  
2. `총자산 = 주문가능현금 + 현재 보유 평가금액` 계산  
3. `최대 투자 가능액 = 총자산 * (1 - MIN_CASH_RATIO) - 현재 보유 평가금액` 계산  
4. `주문가능현금 * BUY_PCT`를 기준으로 주문 금액 산정  
5. `BUY_MIN_AMOUNT` 이상, `BUY_MAX_AMOUNT` 이하 + 최대 투자 가능액 한도 보정  
6. `qty = order_amount // 현재가`로 최종 수량 결정

즉, 고정 금액이 아니라 **계좌 상태에 연동된 동적 포지션 크기**이며,  
동시에 **현금 30% 이상 유지(투자 70% 이내)** 제약이 항상 적용됩니다.

## 6) 손절/물타기 로직

- 손절선 하회 시 `STOP_BREACH_CONFIRM_COUNT` 연속 확인 전에는 대기
- 연속 하회가 확인되면:
  - 아직 물타기 전이라면 1회 물타기 시도
  - 물타기 조건 미충족/실패 시 즉시 손절
- 물타기 조건:
  - `vol_tnrt >= DCA_MIN_VOL_TNRT`
  - `prdy_ctrt >= DCA_MIN_CHANGE_RATE` (급락 구간 회피)
- 물타기 성공 시:
  - 평단/수량 재계산
  - 목표가 재계산
  - 손절가를 `POST_DCA_STOP_LOSS_RATE` 기준으로 더 타이트하게 재설정

## 8) 주요 파라미터 (`config.py`)

| 항목 | 의미 |
|------|------|
| `MAX_STOCKS` | 동시 보유 최대 종목 수 |
| `BUY_PCT` | 종목당 매수 비율 (주문 가능 현금 기준) |
| `BUY_MIN_AMOUNT` | 종목당 최소 주문 금액 |
| `BUY_MAX_AMOUNT` | 종목당 최대 주문 금액 |
| `MIN_CASH_RATIO` | 최소 현금 비중 (예: 0.30 = 30%) |
| `TARGET_RATE_1` | 1차 익절 비율 |
| `TARGET_RATE_2` | 2차 익절 비율 |
| `STOP_LOSS_RATE` | 손절 비율 |
| `POST_DCA_STOP_LOSS_RATE` | 물타기 후 손절 비율 |
| `DCA_MIN_VOL_TNRT` | 물타기 최소 거래량 회전율 |
| `DCA_MIN_CHANGE_RATE` | 물타기 허용 최소 등락률 |
| `STOP_BREACH_CONFIRM_COUNT` | 손절 연속 하회 확인 횟수 |
| `SCORE_THRESHOLD` | 매수 최소 점수 |
| `DEFAULT_MARKET` | 시작 시장 (`KR`/`US`) |
| `US_EXCHANGE` | 미장 거래소 코드 (`NASD`, `NYSE`, `AMEX`) |
| `US_UNIVERSE` | 미장 스크리닝 대상 티커 목록 |
| `SELL_ON_INTERRUPT` | `Ctrl+C` 종료 시 전량 강제매도 여부 |
| `TELEGRAM_NOTIFY_ENABLED` | 텔레그램 상태 알림 사용 여부 |
| `TELEGRAM_NOTIFY_INTERVAL_SEC` | 장중 텔레그램 상태 알림 주기(초, 권장 180) |
| `TELEGRAM_BOT_TOKEN` | BotFather에서 발급받은 텔레그램 봇 토큰 |
| `TELEGRAM_CHAT_ID` | 알림 받을 본인 텔레그램 chat_id |

## 9) 권장 운영 순서

1. 메뉴 `1`로 후보 확인  
2. 후보가 괜찮으면 메뉴 `2` 또는 `3`으로 매수  
3. 장중에 메뉴 `4`를 주기적으로 실행해 익절/손절 반영  
4. 종료 전 메뉴 `5`로 상태 확인  
5. 필요하면 메뉴 `6`으로 리스크 정리

## 10) 트러블슈팅

- 주문 가능 금액 조회 실패:
  - `APP_KEY`, `APP_SECRET`, `ACCOUNT_NO` 형식 확인
  - 모의/실전(`IS_VIRTUAL`) 설정과 계정 권한 확인
- 매수 수량이 0 또는 스킵:
  - 현재가 대비 주문금액이 부족한 상태
  - `BUY_MIN_AMOUNT`, `BUY_MAX_AMOUNT`, `BUY_PCT`, `MIN_CASH_RATIO` 재조정
- 물타기가 자주 스킵됨:
  - `DCA_MIN_VOL_TNRT`가 너무 높거나 `DCA_MIN_CHANGE_RATE`가 너무 보수적일 수 있음
- 종목이 안 뽑힘:
  - `SCORE_THRESHOLD` 또는 등락률/거래량 기준이 너무 엄격할 수 있음

## 11) 주의사항

- 종목 추천과 자동매매는 수익을 보장하지 않습니다.
- 반드시 모의투자에서 충분히 검증한 뒤 실전으로 전환하세요.
- API 키/시크릿 평문 저장은 금지하고, 반드시 `ENC(...)` + `TRADING_MASTER_KEY` 방식으로 관리하세요.
- 현재 구조는 수동 메뉴 기반이므로, 리스크 관리 반영을 위해 장중 메뉴 `4`를 주기적으로 실행해야 합니다.

## 12) 텔레그램 매도 명령

- `TELEGRAM_NOTIFY_ENABLED=True` 이고 봇 토큰/채팅 ID가 설정되어 있으면 텔레그램 명령을 사용할 수 있습니다.
- `/` 또는 `/help` 입력 시 사용 가능한 명령어 목록이 표시됩니다.
- 텔레그램 슬래시 메뉴 노출용 영문 명령도 함께 지원합니다.
  - `/realtime`, `/manager`, `/holdings`, `/sell`, `/sellall`, `/cancel`
- `/실시간`(또는 `/realtime`) 입력 시 3분 주기와 무관하게 장중 브리핑을 즉시 1회 전송합니다.
- `/매니저`(또는 `/manager`) 입력 시 보유 종목별 액션 가이드(보유/비중축소/손절), 신뢰도 점수, 트리거 가격(이탈 시 손절/회복 시 보유)을 표시합니다.
- `/보유`(또는 `/holdings`) 입력 시 총자산(추정), 투자금/현금 비중, 평가손익, 보유 종목별 목표가(1차/2차)/손절가가 표시됩니다.
- `/매도`(또는 `/sell`) 입력 시 보유 종목이 번호로 표시됩니다.
- 종목 번호 선택 후 매도 비율을 선택합니다.
  - `1` = 25%
  - `2` = 50%
  - `3` = 75%
  - `4` = 100%(전량)
- `/전량매도`(또는 `/sellall`) 입력 시 아래 확인 단계 후 전체 종목 전량 매도합니다.
  - `1` = yes
  - `2` = no(취소)
- 진행 중 취소는 `/취소`
- 목표가 도달 시 아래 알림이 발송됩니다.
  - `1차 목표입니다. 매도 권장 알림입니다.`
  - `2차 목표입니다. 매도 권장 알림입니다.`
